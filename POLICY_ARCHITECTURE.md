## Summary

This architecture provides:
- **Admin Control**: Toggle policy per society via UI
- **Safe Defaults**: OFF by default, requires explicit enablement
- **Non-Destructive**: Doesn't replace existing verification
- **Optional Enforcement**: Middleware can be applied selectively
- **Backward Compatible**: No breaking changes
- **Future-Ready**: Easy to enforce when needed
